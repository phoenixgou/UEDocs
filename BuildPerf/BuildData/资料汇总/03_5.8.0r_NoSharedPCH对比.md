## 讨论范围

- 检查 `D:\UE\5.8.0r\Engine\Saved\UnrealBuildTool\BuildConfiguration.xml` 中 `<bUseSharedPCHs>false</bUseSharedPCHs>` 是否生效。
- 用显式 `-NoSharedPCH` 执行 clean/build，验证 `D:\UE\5.8.0r` 的 NoSharedPCH 构建。
- 注释掉 `bUseSharedPCHs=false` 后再次 clean/build，验证 SharedPCH enabled 状态。
- 分析 `D:\UE\5.8.0r\Engine\Intermediate\Build\XGETasks.xml`、IncrediBuild Build42/43、remote task、`.rsp`、`/Yu`、`/Fp`。
- 将 `D:\UE\5.8.0r` 证据合并到 `D:\UE\Docs\Build\SharedPCH数学证明\SharedPCH冷态分发税数学证明.md`。

## 关键结论

1. Build42 是 NoSharedPCH 成功样本，`SharedPCHToolNodes=0`，构建后 `SharedPCHCount=0`。
2. Build43 是 SharedPCH enabled 完整执行样本，但 IncrediBuild/XGE 最终出现 `KERNELBASE.dll` access violation；它可用于冷态分发成本分析，但不是一次干净成功样本。
3. Build43 的 SharedPCH 任务全部在本机 `GIH-D-33028` 成功执行；慢主要体现在 remote helper 首批使用 SharedPCH 的任务拉长。
4. Build42 与 Build43 的 Send/Recv、remote `cl.exe` 平均耗时和 high-send span 与 `T_warmup ~= H * S / B` 模型量级闭合。
