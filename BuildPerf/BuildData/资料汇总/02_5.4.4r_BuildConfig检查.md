## 讨论范围

- 检查 `D:\UE\5.4.4r\Engine\Saved\UnrealBuildTool\BuildConfiguration.xml` 与用户级 UBT 配置是否启用 `NoSharedPCH`。
- 分析 UE 5.4.4 在 MSVC 14.44 下暴露的 `__has_feature`、`INFINITY`、`chrono`、`C4702`、`C4756` 等编译问题。
- 安装并固定 MSVC `14.38.33130`，使 `D:\UE\5.4.4r` 使用 UE 5.4 Windows SDK JSON 推荐工具链。
- 修复 `D:\UE\5.4.4r\Engine\Source\Runtime\Core\Public\Experimental\ConcurrentLinearAllocator.h` 的 `defined(__has_feature)` guard。
- 修复 4 个 `INFINITY` 触发 `C4756` 的点，并用单文件编译验证。
- 形成 Build39 NoSharedPCH 成功、Build40 SharedPCH enabled 失败、Build41 SharedPCH 增量成功的对照。

## 关键结论

1. `D:\UE\5.4.4r\Engine\Saved\UnrealBuildTool\BuildConfiguration.xml` 后续包含 `<bUseSharedPCHs>false</bUseSharedPCHs>`，NoSharedPCH 配置在 Build39 生效。
2. MSVC 14.44 暴露了 UE 5.4.4 源码兼容问题；固定 MSVC `14.38.33130` 是更符合 `D:\UE\5.4.4r\Engine\Config\Windows\Windows_SDK.json` 的路线。
3. Build39 是 NoSharedPCH 成功基准；Build40 是 SharedPCH enabled 后的失败样本，不应被当作成功基准。
4. Build40 虽失败，但它的 remote task、流量和 PCH 产物能作为 SharedPCH 冷态分发税样本。
